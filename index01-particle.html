Automate, reconcile & transactReconMate is an automated reconciliation & transaction matching application developed to automate typical routine reconciliation of transactions.

Built to automatically peruse and identify transactions from a master data source and match them with third party systems that interact with the master data. The application allows our clients to configure/map the master file as well as third party data sources. The application is powered by an analytics engine that is able to analyse transactions and present them in desired visualisations based on our clients’ needs.

More Info

The application is configurable to suit any unique requirements of our clients. It is built to eliminate any significant software development for our clients. While it can be fully automated, it provides capability for semi-automation as described below:


Full automation

This involves integrating APIs to the system from the core banking system for source data mapping, and third party APIs for transaction reconciliation and matching with the core banking data.

Semi-automation

This will involve development of a Secure File Transfer Protocol (SFTP), that will allow for the files downloaded from the various systems securely sent to a server outside the bank’s environment. Our application will then be able to access the files and perform the reconciliation and transaction matching from this secure environment.

Manual

This involves uploading of files extracted from each of the system i.e. source data from the core banking system and transaction data from third party systems. These files are then uploaded on the application with does the reconciliation and mapping.

Implementation

ReconMate can be deployed on the cloud or on premises depending on the needs of the client.

We recommend the fully automated version of the system to benefit from the full value proposition of the application. Should you not be able to take up the fully automated version, we would be able to implement the semi-automated version, and then work with you and your technology team to build the required capacity to deploy the superior version.

Cloud Based

This will make the application available to the bank’s team through the web. The bank’s technology leadership team will assign and manage credentials for the application. Nouveta will be fully responsible for security, hosting, server space and any other functionality that is required to keep the application in proper working order. The bank will provide details of users to be provided credentials to access the system.

On Premise

The application will be installed on the bank’s environment and will be managed by the bank’s technology team. The bank will be responsible for security, hosting and server space for the application.
